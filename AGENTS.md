# Repository Guidelines

## Project Structure & Module Organization

This is a small Rust 2024 crate for `nullidentd`, a dummy RFC 1413 ident daemon. The binary entry point is `src/bin/nullidentd.rs`; shared exports live in `src/lib.rs`; command-line and logging configuration live in `src/config.rs`. Build metadata is generated by `build.rs`. There is no `tests/` tree yet, so place narrow unit tests beside the module they exercise and add integration tests under `tests/` for external daemon behavior.

## Build, Test, and Development Commands

- `cargo build`: compile the debug build.
- `cargo build --release`: compile the optimized release binary.
- `cargo build --profile minsize`: compile the size-optimized profile defined in `Cargo.toml`.
- `cargo run -- --listen localhost:1113 --ident nobody`: run locally with explicit listen address and ident response.
- `cargo test`: run all unit and integration tests.
- `cargo fmt`: apply the repository rustfmt settings.
- `cargo clippy --all-targets --all-features`: run lint checks before submitting nontrivial changes.
- `cargo outdated --root-deps-only`: check whether direct dependencies can be updated.

## Coding Style & Naming Conventions

Use standard Rust naming: `snake_case` for functions, modules, and variables; `UpperCamelCase` for types; `SCREAMING_SNAKE_CASE` for constants such as `BUF_SZ`. Follow `rustfmt.toml`: 120-column maximum width, crate-granular imports, and grouped imports. Keep async networking code explicit and readable.

## Testing Guidelines

Use Rust's built-in test framework. Unit-test pure behavior such as option parsing and log-level selection in the relevant source file. For daemon behavior, prefer integration tests that bind to an ephemeral local port, send an ident query, and assert the fixed `USERID : UNIX : <ident>` response.

## Commit & Pull Request Guidelines

Recent history uses short, imperative commit subjects, often like `cargo update`. Keep subjects concise, for example `add timeout integration test`. Pull requests should include a behavior summary, commands run, and any operational impact such as changed defaults, listen behavior, logging, or release profile settings.

## Security & Configuration Tips

Do not expose private user identities through the default response. The daemon intentionally returns a configured static ident string; keep examples and defaults privacy-preserving. Treat default listen addresses and timeouts as operationally sensitive.

## Dependency Maintenance

Direct dependencies are declared in `Cargo.toml`; the checked-in lockfile records the resolved graph. Prefer narrow dependency changes in their own commit, run `cargo update` only when intentionally refreshing the lockfile, and verify with `cargo test`.
